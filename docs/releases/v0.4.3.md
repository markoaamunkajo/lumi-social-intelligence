# Lumi Social Intelligence v0.4.3

**Status:** public release
**Release theme:** Live Surface readiness for Gateway-start warmed context, written pre-tool acknowledgement, and no silent expensive tool waits.

## Summary

v0.4.3 is a Live Surface readiness release. It tightens the boundary between fast, safe instant controls and longer tool paths:

- Live Surface tools that belong on the surface must be ready after Gateway start, not lazily warmed on the first user need.
- Host setups should bind the public-safe prefill file with `prefill_messages_file: ~/.hermes/state/live_surface_prefill_messages.json`.
- Simple familiar checks get a tiny written acknowledgement before the tool path opens.
- Longer or expensive AutoResearch/tool paths must write a concrete acknowledgement before the user is left waiting.
- Emoji reactions are **not** acknowledgements for longer or expensive tool paths; the contract switches from reaction to writing when a tool call is expensive.

## Why this patch exists

v0.4.2 made instant reactions explicit: already-warmed reaction targets should resolve with zero tool calls and no DB archaeology. v0.4.3 closes the other side of that rule. When the next step is a longer or expensive tool path, a reaction is too small and too ambiguous; Lumi must write a brief handoff instead. The public release also makes the host startup binding explicit: the Live Surface prefill messages file should be wired through the top-level Hermes config key `prefill_messages_file`, pointing at `~/.hermes/state/live_surface_prefill_messages.json`, so behaviour priming exists before the first user need.

That prevents reaction-heavy behavior from masking latency or making the user wonder whether work is happening.

## Not claimed

- The public repository sends Telegram messages or reactions itself.
- The public repository promotes a private Hermes Live Surface at runtime.
- Emoji reactions are acceptable acknowledgements for longer or expensive tool calls.
- First-use warmup in the user path is acceptable for Live Surface tools.
