# Lumi Social Intelligence v0.2.0

Release-candidate notes for **Lumi Social Intelligence v0.2.0**: the first shareable demo package for the context → appraisal → Presence decision → receipt path.

This is still a deliberately cautious release-candidate. It packages the public-safe demo evidence, verifies the demo contract, and keeps live Telegram/native reaction claims out until the repository contains observed live evidence.

## Observed / verified in this repository

The v0.2.0 release-candidate verifies a receipt-first demo path using synthetic/public-safe context:

```text
synthetic/public-safe context → appraisal → Presence decision → review card → demo receipt
```

Included evidence:

- `docs/demos/v0.2-demo-evidence.json`
- `docs/demos/v0.2-demo-side-by-side.json`
- `docs/demos/v0.2-demo-side-by-side.md`
- `docs/demos/v0.2-demo-index.md`
- `docs/demos/v0.2-demo-script.md`
- `scripts/verify_v02_demo_package.py`

The verifier proves that required demo receipts exist, safety counters are present, blocked-action explanations are present, the Markdown matches the JSON boundary, and no unsupported live native reaction claim is present.

## Review-gated / no-write boundary

The demo remains non-mutating and review-gated:

```text
canonical_writes: 0
runtime_actions: []
telegram_messages_sent: 0
telegram_reactions_sent: 0
requires_human_review_before_live_wiring: true
```

This means the release-candidate can show the product contract honestly without writing durable memory, sending Telegram messages, sending Telegram reactions, reading private scheduler state, or storing raw private transcripts.

## Not yet supported or not yet claimed live

The side-by-side evidence keeps the current boundary explicit:

| Behavior | v0.2.0 RC status |
|---|---|
| Review card generated from synthetic context | observed in this repository |
| Context/appraisal/Presence decision receipt | observed in this repository |
| native outbound emoji reaction delivery | `not_claimed` / shadow-only |
| Telegram send/reaction side effects | not supported by this release-candidate |
| Durable memory promotion / canonical host writes | not supported by this release-candidate |

Native outbound reaction delivery is **not claimed live** by this repository yet. Any future release may claim it only when the demo receipt contains observed evidence for that behavior.
