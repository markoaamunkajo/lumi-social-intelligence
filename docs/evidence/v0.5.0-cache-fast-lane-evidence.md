# Lumi Social Intelligence v0.5.0 cache-backed fast-lane evidence

**Status:** `verified`<br>
**Version:** `0.5.0`<br>
**Principle:** A narrow, authorized local-cache reply may skip agent work; every uncertainty falls through normally<br>
**Claim boundary:** Public-safe contract/evidence only; no private adapter, cache contents, identifiers, Telegram sends, or runtime promotion

v0.5.0 defines a deliberately narrow promotion path for a private host adapter. A host may promote an approved, cache-backed tool capability to Live Surface so an eligible request can receive a local reply without opening the agent/tool path. The public repository does not ship that adapter or the cache.

## Eligibility and cache boundary

| Field | Value |
|---|---|
| Authorized sender required | `True` |
| Platform scope | `telegram_direct_message` |
| Host-approved Live Surface capability required | `True` |
| Fresh + valid cache required | `True` |
| Cache location | `host_local_state_only` |
| Network I/O allowed | `False` |
| Cache ceiling | `12-hour cache freshness ceiling` |
| Raw cache content exported | `False` |

## Dispatch boundary

- Authorization precedes direct reply: `True`.
- Agent session/compression allowed: `False`.
- Agent tool calls allowed: `0`.
- Reply character limit: `4096`.
- Delivery still uses the host's normal adapter: `True`.

## Fail closed

**Falls through normally** when the sender is unauthorized, the platform is not a Telegram DM, the capability is not host-approved, or the local cache is missing, invalid, or stale.

| Condition | Required host action |
|---|---|
| Unauthorized sender | `normal_auth_or_pairing_flow` |
| Non-DM / unsupported platform | `normal_full_dispatch` |
| Unapproved or missing Live Surface capability | `normal_full_dispatch` |
| Missing, invalid, or stale cache | `normal_full_dispatch` |
